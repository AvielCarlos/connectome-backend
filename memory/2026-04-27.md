### Major Architecture Additions Deployed

**Agent Evolution System (Ora Self-Evolution)**
- `ora/agent_registry.py` — persistent agent registry in Redis (ora:agent_registry), tracks name/type/weight/status/lineage/performance
- `ora/agents/evolution_agent.py` — OraEvolutionAgent: partition, merge, spawn, cannibalize, retire
- Population fitness scoring: avg_rating × (1 - skip_rate) × (1 + save_rate)
- Partition: high-performing agents with high variance → split into 2 specialists
- Merge: two low-performing overlapping agents → one hybrid
- Cannibalize: absorb failing agent's best patterns into a thriving one, retire the weak
- Spawn: detect user need gap, generate new agent, start at 5% trial weight
- Retire: consistently failing agents removed from gene pool
- Builtin agents NEVER auto-retired without Avi's explicit approval
- Evolution events logged to ora_lessons with full context
- New API endpoints: GET/POST /api/ora/evolution/population, /history, /run, /lineage/{name}
- Profile page System tab updated with Agent Population card (green/yellow/red fitness)
- Wired into autonomy_agent.py as step F (every 6h)
